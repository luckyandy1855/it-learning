# LeetCode 解题文档系统设计

**日期**：2026-06-17
**状态**：待实施

---

## 背景与目标

为个人算法学习建立一套结构化的解题文档系统。按 LeetCode 题号顺序，批量生成每题一个 Markdown 文件，供本地仓库和 Notion 预览使用。每篇文档由 AI 辅助生成（题目转述、思路分析、按题型匹配的代码实现、复杂度、踩坑），用户再对照官方题目和提交结果校验。

每篇题解优先保证：**读者不用打开原题也能看懂题目要求**，同时避免机械复制题面；题目描述使用中文完整转述，示例和关键约束需保留。

---

## 目录结构

```
leetcode/
├── _template.md           # 标准模板（人工参考，不参与生成）
├── README.md              # 总索引表：题号 | 题名 | 难度 | 标签
├── 0001-0100/             # 第 1-100 题
│   ├── 0001-two-sum.md
│   └── ...
├── 0101-0200/             # 第 101-200 题
├── 0201-0300/
└── ...                    # 每 100 题一个子目录，依此类推
```

**子目录命名规则**：`{起始题号4位}-{结束题号4位}/`，如 `0001-0100/`、`0501-0600/`

**文件命名规则**：`{4位题号补零}-{题名-kebab-case}.md`

示例：`leetcode/0001-0100/0001-two-sum.md`、`leetcode/0001-0100/0020-valid-parentheses.md`

**批次规划**：每次生成 50 题，一个子目录对应两个批次。每批完成后在执行计划中标记 `✅ Done`。

**专题索引**：后续按标签建立专题索引，独立存放，不影响题号分段目录结构。

> **说明**：`leetcode/` 是个人算法知识库目录，独立于 `content/`（内容选题）、`data/`（运营数据），不属于内容创作流程。需同步在 `docs/项目目录结构.md` 中补充说明。

## 元数据规范

每篇文件保留一个轻量元数据块：

```yaml

难度: 简单
```

**难度取值**：简单 / 中等 / 困难（须以 LeetCode 官方页面为准，AI 不凭记忆补全）

**标签取值**：须以 LeetCode 官方题目标签为准，AI 不凭记忆补全

> 不在单篇文档里放 `标签来源`、`日期`、`状态` 等维护字段，避免同步到 Notion 后开头过重。验证、提交状态后续如需管理，放到总索引或单独数据表中。

## 注释规范

- **函数头注释**：说明函数功能、参数含义、返回值
- **关键逻辑注释**：解释"为什么这样写"，而非"写了什么"
- **边界注释**：标注特殊输入的处理方式（空数组、单元素、负数等）

三种语言遵循各自惯用注释风格：Go 用 `//`，Java 用 `/** */` + `//`，Python 用 `#`。Java 示例如使用 `Map`、`HashMap`、`List`、`Queue` 等标准库，需补全必要 import，方便复制到本地验证。

### 题型例外

不是所有 LeetCode 题都必须提供 Go / Java / Python 三语言实现。代码实现应优先匹配题目在 LeetCode 上的真实提交形态：

- **普通算法题**：默认提供 Go、Java、Python 三种语言实现，且代码块应尽量可复制到 LeetCode 对应语言环境中运行。
- **数据库题**：使用 `SQL` / `MySQL` 作为主解法；不生成 Go、Java、Python 空壳或 TODO 占位。
- **Shell 题**：使用 `Bash` / `Shell` 作为主解法；不生成 Go、Java、Python 空壳或 TODO 占位。
- **交互题、系统设计题或平台特定 API 题**：按 LeetCode 提供的函数签名和主流提交语言选择实现；如果某语言没有合理签名，应省略该语言，而不是强行补模板。
- **源数据缺少某语言实现时**：优先补一个可读、可解释、与题意一致的实现；无法可靠补全时，省略该语言并说明原因，不保留 `TODO`。

## 生成流程

1. 用户指定批次范围（如"生成第 1-20 题"）
2. Claude 按题号顺序逐题生成 `.md` 文件，元数据中难度/标签需用户自行对照 LeetCode 官方页面校准，AI 不凭记忆填写
3. 题目描述需完整转述题意、示例和约束，确保离开官方页面也能看懂题目
4. 代码实现按题型选择语言：普通算法题默认 Go / Java / Python；数据库题用 SQL；Shell 题用 Bash；其他特殊题按 LeetCode 实际提交形态处理
5. 同步在 `README.md` 追加对应索引行
6. 用户在 LeetCode 上提交验证代码；验证状态不写入单篇正文，避免文档尾部出现空表格或占位信息

---

## 约束与说明

- **题目描述**：不机械复制 LeetCode 题面；用中文完整转述题目要求，保留示例数据和关键约束，保证复习时不需要频繁跳转原题
- **元数据准确性**：难度、标签字段须以 LeetCode 官方页面为准，AI 生成后用户校验，不凭记忆补全
- **代码验证**：AI 生成代码不等于正确代码，用户须在 LeetCode 或本地环境验证；验证状态不放进单篇题解正文
- **代码语言选择**：普通算法题默认三语言实现；数据库、Shell、交互或平台特定题按题型选择主语言，不生成无意义的三语言占位
- **示意图**：优先使用 ASCII 表格/结构图，避免 Mermaid 在 Notion 中渲染成代码块或占用过大版面
- **工具**：不引入任何构建工具或脚本，纯 Markdown 文件管理
- **目录说明**：`leetcode/` 目录需同步更新至 `docs/项目目录结构.md`
