# AGENTS.md

## 文件用途

本仓库是一个以学习资料为核心的个人知识库，当前主要包含：

- `leetcode/`：LeetCode 题解与索引
- `前端从入门到AI应用开发/`：前端与 AI 应用开发学习笔记
- `docs/`：过程文档、计划、说明
- `templates/`：可复用模板

本文件同时服务于人类维护者和 AI 代理，目标是让协作过程更安全、更稳定、更容易审查。

## 语言约定

1. `AGENTS.md` 与 `MEMORY.md` 后续统一使用中文维护。
2. 仓库中的学习型内容，如无特殊要求，优先保持中文表达。
3. 新增说明文档时，优先使用简洁、明确、可执行的中文。

## 目录说明

- `README.md`：仓库入口说明
- `leetcode/`：LeetCode 题解文档、索引、辅助脚本
- `templates/`：共享模板
- `docs/`：规格、计划、过程记录
- `前端从入门到AI应用开发/`：课程式主题笔记

## 协作原则

1. 优先进行小而安全的结构化改动。
2. 除非用户明确要求重组，否则尽量保持既有命名方式。
3. 不要在未确认的情况下批量删除、覆盖或重写学习内容。
4. 移动文件时，如改动范围允许，应同步更新明显可见的引用路径。
5. 避免把 `.DS_Store` 一类系统噪音文件纳入版本管理。

## LeetCode 区域约定

- 题解文件按题号区间放在子目录中，例如 `leetcode/0001-0100/`
- 文件名延续现有格式：`0001-two-sum.md`
- 当前索引文件位置：
  - `leetcode/index-by-difficulty.md`
  - `leetcode/index-by-topic.md`
- 当前辅助脚本位置：
  - `leetcode/build_leetcode_indexes.js`
  - `leetcode/calibrate_leetcode_metadata.js`
  - `leetcode/generate_leetcode_batch.js`
  - `leetcode/repair_leetcode_special_topics.js`
- 当前共享题解模板位置：
  - `templates/leetcode-problem-template.md`

## 文档维护规则

1. 学习内容应优先保证可读性和结构稳定性。
2. 如果某次改动涉及路径迁移，应尽量同步修正文档中的旧路径。
3. 过程文档可以简短，但结论、路径、约定应写清楚。
4. 如果形成新的仓库级约定，应补充到 `AGENTS.md` 或 `MEMORY.md`。

## 改动边界

- 可以直接进行的改动：
  - 新增或完善学习笔记
  - 在用户明确要求下调整目录结构
  - 同步修改辅助脚本及相关说明
  - 补充轻量级项目文档
- 需要额外谨慎的改动：
  - 批量重命名
  - 脚本驱动的大规模生成
  - 会影响大量文件的索引重建
- 未经明确许可应避免的改动：
  - 破坏性清理
  - 大范围内容重写
  - 删除重要主题目录

## 最低校验要求

在声称任务完成之前，至少检查：

- 目标文件是否位于预期路径
- 明显引用是否仍然有效
- `git status --short` 是否符合预期变更范围

如果脚本被移动或重命名，应额外检查相关使用说明中的路径是否同步更新。

## 协作备注

1. 做结构调整前，先说明准备修改什么。
2. 如存在多个合理命名方案，优先选择语义更清晰的名字。
3. 仓库级的重要事实要及时记录到 `MEMORY.md`，避免后续重复摸索。
