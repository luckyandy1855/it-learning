# 条件语句（JS-007）

## 🎯 本节学习目标

- 掌握 `if`、`else if`、`else` 的基本写法。
- 理解条件语句如何根据不同情况执行不同代码。
- 能够用条件语句处理 AI 应用中的空输入、加载中和错误状态。

---

## 📖 什么是条件语句

条件语句让程序具备“根据情况做不同事情”的能力。没有条件语句，代码只能从上到下机械执行；有了条件语句，程序可以判断用户输入、请求状态和接口结果。

在 AI 聊天应用中，条件语句非常常见。例如 Prompt 为空时不发送，正在生成时禁用重复提交，接口失败时显示错误消息，模型不同则使用不同参数。

`if` 是最常用的条件语句。它会判断括号里的表达式是否为真，如果为真，就执行对应代码块。

---

## 🧠 原理讲解

条件语句的核心结构是：

```text
如果 条件成立：
  执行 A
否则：
  执行 B
```

对应 JavaScript：

```js
if (condition) {
  // 条件为 true 时执行
} else {
  // 条件为 false 时执行
}
```

当有多个分支时，可以使用 `else if`：

```js
if (status === "loading") {
  console.log("生成中");
} else if (status === "error") {
  console.log("生成失败");
} else {
  console.log("可以发送");
}
```

---

## 🏗 基本结构

```html
<script>
  const prompt = "";

  if (prompt.trim() === "") {
    console.log("请输入 Prompt");
  } else {
    console.log("可以发送");
  }
</script>
```

## ✅ 完整代码

下面用条件语句处理 AI Prompt 的发送状态：

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>条件语句示例</title>
</head>
<body>
  <h1>AI Prompt 状态判断</h1>
  <textarea id="promptInput" placeholder="输入 Prompt"></textarea>
  <button id="sendButton" type="button">发送</button>
  <p id="status">等待输入</p>

  <script>
    const promptInput = document.querySelector("#promptInput");
    const sendButton = document.querySelector("#sendButton");
    const status = document.querySelector("#status");

    let isGenerating = false;

    sendButton.addEventListener("click", function () {
      const prompt = promptInput.value.trim();

      if (isGenerating) {
        status.textContent = "AI 正在生成，请稍后再试。";
      } else if (prompt === "") {
        status.textContent = "Prompt 不能为空。";
      } else if (prompt.length > 200) {
        status.textContent = "Prompt 太长，请控制在 200 字以内。";
      } else {
        isGenerating = true;
        status.textContent = "已发送，正在等待 AI 回复：" + prompt;
      }
    });
  </script>
</body>
</html>
```

---

## 🔍 逐行解析

`let isGenerating = false;` 保存当前是否正在生成回复。

`if (isGenerating)` 优先判断是否处于生成中，防止重复发送。

`else if (prompt === "")` 判断输入是否为空。

`else if (prompt.length > 200)` 判断输入是否超出长度限制。

`else` 表示前面条件都不成立，此时可以发送 Prompt。

`isGenerating = true;` 发送后把状态改为生成中。

---

## 🌐 浏览器表现

页面打开后显示“等待输入”。如果直接点击发送，会提示 Prompt 不能为空。

输入超过 200 字后点击发送，会提示 Prompt 太长。输入合法内容后点击发送，会显示正在等待 AI 回复。再次点击时，会提示 AI 正在生成。

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `if` | 条件语句 | 条件为真时执行代码块 | `if (ok) {}` |
| `else if` | 条件分支 | 增加更多判断分支 | `else if (error) {}` |
| `else` | 默认分支 | 前面条件都不满足时执行 | `else {}` |
| `===` | 比较运算符 | 严格相等判断 | `prompt === ""` |
| `length` | 字符串属性 | 获取字符串长度 | `prompt.length` |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 条件语句让程序根据不同状态执行不同逻辑。
- 多个分支有先后顺序，越关键的判断越应该靠前。
- AI 发送逻辑通常要先判断生成中，再判断输入合法性。

---

## ⚠️ 易错点

- 易错点1：把所有判断都写成独立 `if`。正确写法：互斥条件使用 `if / else if / else`。
- 易错点2：把空输入判断放在发送逻辑之后。正确写法：先校验，再发送。
- 易错点3：忘记处理默认情况。正确写法：使用 `else` 兜底。

---

## 💡 最佳实践

- 条件分支按优先级排列，先处理不能继续执行的情况。
- 分支里的提示语要具体，让用户知道如何修正。
- 复杂条件可以提前拆成布尔变量，提升可读性。

## 🚀 AI 应用场景

真实 AI 应用发送前常见判断：

```js
if (isGenerating) {
  showMessage("正在生成中");
} else if (prompt.trim() === "") {
  showMessage("请输入 Prompt");
} else if (!selectedModel) {
  showMessage("请选择模型");
} else {
  sendPrompt(prompt);
}
```
