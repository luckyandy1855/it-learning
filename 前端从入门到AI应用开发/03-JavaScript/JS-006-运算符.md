# 运算符（JS-006）

## 🎯 本节学习目标

- 掌握 JavaScript 中常见运算符的用途。
- 理解算术、比较、逻辑和赋值运算符的区别。
- 能够用运算符处理 AI 应用中的输入校验、参数计算和状态判断。

---

## 📖 什么是运算符

运算符是 JavaScript 中用来对值进行操作的符号或关键字。它可以做数字计算、字符串拼接、大小比较、逻辑判断和变量赋值。

在 AI 应用中，运算符很常见。例如判断 Prompt 是否为空、计算剩余 token 数、根据开关决定是否启用流式输出、把用户名和输入内容拼成展示文本。

运算符不是孤立知识点，它是后续条件语句、循环、函数和接口请求的基础。

---

## 🧠 原理讲解

表达式是可以产生一个值的代码片段，运算符负责把多个值组合成新的结果。

```js
const remainingTokens = 8000 - usedTokens;
```

这行代码里，`8000 - usedTokens` 是表达式，`-` 是运算符，最终结果会赋值给 `remainingTokens`。

常见运算符可以按用途分组：

```text
算术运算符：+ - * / %
比较运算符：=== !== > < >= <=
逻辑运算符：&& || !
赋值运算符：= += -=
```

## ✅ 完整代码

下面用运算符检查 Prompt 是否可以发送：

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>运算符示例</title>
</head>
<body>
  <h1>AI Prompt 校验</h1>
  <textarea id="promptInput" placeholder="输入不超过 100 字的 Prompt"></textarea>
  <button id="checkButton" type="button">检查</button>
  <p id="result"></p>

  <script>
    const promptInput = document.querySelector("#promptInput");
    const checkButton = document.querySelector("#checkButton");
    const result = document.querySelector("#result");
    const maxLength = 100;

    checkButton.addEventListener("click", function () {
      const prompt = promptInput.value.trim();
      const length = prompt.length;
      const isNotEmpty = length > 0;
      const isWithinLimit = length <= maxLength;
      const canSend = isNotEmpty && isWithinLimit;
      const remaining = maxLength - length;

      if (canSend) {
        result.textContent = "可以发送，还剩 " + remaining + " 个字符。";
      } else {
        result.textContent = "暂不能发送，请检查是否为空或超出长度限制。";
      }
    });
  </script>
</body>
</html>
```

---

## 🔍 逐行解析

`const maxLength = 100;` 定义 Prompt 最大长度限制。

`const prompt = promptInput.value.trim();` 读取输入并去掉首尾空白。

`const length = prompt.length;` 得到字符串长度。

`length > 0` 使用比较运算符判断 Prompt 是否非空。

`length <= maxLength` 判断输入是否没有超过长度上限。

`isNotEmpty && isWithinLimit` 使用逻辑与运算符，只有两个条件都为 `true` 时，结果才是 `true`。

`maxLength - length` 使用算术运算符计算剩余字符数。

---

## 🌐 浏览器表现

输入 1 到 100 个字符并点击检查，页面会提示可以发送，并显示剩余字符数。

如果输入为空或超过 100 个字符，页面会提示暂不能发送。这个逻辑与真实 AI 产品中的 Prompt 输入校验很接近。

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `+` | 算术/拼接运算符 | 数字相加或字符串拼接 | `"AI" + "助手"` |
| `-` | 算术运算符 | 数字相减 | `100 - length` |
| `===` | 比较运算符 | 严格相等 | `status === "done"` |
| `!==` | 比较运算符 | 严格不相等 | `role !== "user"` |
| `&&` | 逻辑运算符 | 两侧都为真才为真 | `a && b` |
| `||` | 逻辑运算符 | 任一为真即为真 | `a || b` |
| `!` | 逻辑运算符 | 取反 | `!isLoading` |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 比较相等时优先使用 `===` 和 `!==`。
- 逻辑运算符常用于组合多个判断条件。
- AI 应用中的输入校验、状态判断和参数计算都依赖运算符。

---

## ⚠️ 易错点

- 易错点1：使用 `=` 判断相等。正确写法：判断相等用 `===`，赋值才用 `=`。
- 易错点2：把 `&&` 和 `||` 的含义混淆。正确理解：`&&` 要求全部满足，`||` 只要满足一个。
- 易错点3：字符串数字直接计算导致结果不符合预期。正确写法：需要计算时先转成数字。

---

## 💡 最佳实践

- 判断相等时使用严格比较，减少隐式类型转换带来的问题。
- 复杂判断可以拆成有名字的布尔变量，例如 `isNotEmpty`。
- 与用户输入相关的判断，要同时考虑空值、长度和格式。

---

## 🚀 AI 应用场景

发送 Prompt 前通常需要判断输入、模型和请求状态：

```js
const hasPrompt = prompt.trim().length > 0;
const hasModel = selectedModel !== "";
const isReady = !isGenerating;
const canSend = hasPrompt && hasModel && isReady;
```
