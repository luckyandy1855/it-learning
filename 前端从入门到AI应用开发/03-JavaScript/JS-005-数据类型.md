# 数据类型（JS-005）

## 🎯 本节学习目标

- 掌握 JavaScript 中常见基础数据类型和引用数据类型。
- 理解不同数据类型在 AI 应用状态中的用途。
- 能够用合适的数据结构保存 Prompt、消息、配置和请求状态。

---

## 📖 什么是数据类型

数据类型描述一个值是什么种类。JavaScript 中常见基础类型包括 `string`、`number`、`boolean`、`undefined`、`null`、`bigint`、`symbol`。常见引用类型包括对象、数组和函数。

在 AI 应用中，用户输入通常是字符串，温度参数是数字，是否正在生成是布尔值，消息列表通常是数组，单条消息通常是对象。

理解数据类型可以帮助你判断应该如何保存数据、如何校验输入、如何把数据发送给接口，以及如何把接口返回结果渲染到页面上。

---

## 🧠 原理讲解

JavaScript 是动态类型语言。变量本身不固定类型，值才有类型。

```js
let currentValue = "AI 助手";
currentValue = 42;
currentValue = true;
```

这段代码是允许的，因为 `currentValue` 可以先保存字符串，再保存数字和布尔值。但真实项目中不建议随意改变同一变量的含义，否则代码会难以维护。

可以使用 `typeof` 查看大多数基础类型：

```js
typeof "hello"; // "string"
typeof 123; // "number"
typeof true; // "boolean"
```

## ✅ 完整代码

下面用不同数据类型描述一个 AI 请求状态：

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>JavaScript 数据类型示例</title>
</head>
<body>
  <h1>AI 请求数据类型</h1>
  <pre id="output"></pre>

  <script>
    const prompt = "请把这段会议记录整理成三条行动项。";
    const temperature = 0.6;
    const isStreaming = true;
    const selectedModel = "gpt-4.1-mini";
    const messages = [
      {
        role: "user",
        content: prompt
      }
    ];

    const requestBody = {
      model: selectedModel,
      temperature: temperature,
      stream: isStreaming,
      messages: messages
    };

    const output = document.querySelector("#output");
    output.textContent = JSON.stringify(requestBody, null, 2);
  </script>
</body>
</html>
```

---

## 🔍 逐行解析

`const prompt = "...";` 使用字符串保存用户输入。

`const temperature = 0.6;` 使用数字保存生成温度参数。

`const isStreaming = true;` 使用布尔值表示是否启用流式输出。

`const selectedModel = "gpt-4.1-mini";` 使用字符串保存模型名称。

`const messages = [...]` 使用数组保存消息列表，因为对话通常包含多条消息。

数组里的 `{ role: "user", content: prompt }` 是对象，适合保存一条消息的多个字段。

`JSON.stringify(requestBody, null, 2)` 把对象格式化成便于阅读的 JSON 字符串。

---

## 🌐 浏览器表现

页面会显示一个格式化后的 JSON 对象。它包含模型名称、温度、是否流式输出和消息列表。

这类结构与真实 AI API 请求体非常接近，只是示例没有发起网络请求。

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `string` | 基础类型 | 文本数据 | `"你好"` |
| `number` | 基础类型 | 数值数据 | `0.7` |
| `boolean` | 基础类型 | 真或假 | `true` |
| `Array` | 引用类型 | 有序列表 | `["user", "assistant"]` |
| `Object` | 引用类型 | 键值对集合 | `{ role: "user" }` |
| `typeof` | 运算符 | 查看值类型 | `typeof prompt` |
| `Array.isArray()` | API | 判断是否为数组 | `Array.isArray(messages)` |
| `JSON.stringify()` | API | 把对象转成 JSON 字符串 | `JSON.stringify(data)` |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 字符串、数字、布尔值是最常用的基础类型。
- 数组适合保存消息列表，对象适合保存单条消息或配置。
- AI API 请求体本质上就是一组有结构的数据。

---

## ⚠️ 易错点

- 易错点1：把数字写成字符串后直接计算。正确写法：需要计算时使用数字类型，例如 `0.7`，不是 `"0.7"`。
- 易错点2：用多个零散变量保存一条消息。正确写法：使用对象组织 `role` 和 `content`。
- 易错点3：用 `typeof` 判断数组。正确写法：使用 `Array.isArray(messages)`。

---

## 💡 最佳实践

- 同一个变量尽量保持稳定含义，不要一会儿保存字符串，一会儿保存对象。
- 对复杂数据先画出结构，再写代码。
- 与 AI API 交互前，先确认请求体字段类型是否符合接口要求。

## 🚀 AI 应用场景

下面是一段常见 AI 请求体结构。它把用户 Prompt、模型配置和消息列表组合成一个对象。

```js
const aiRequest = {
  model: "gpt-4.1-mini",
  temperature: 0.7,
  stream: true,
  messages: [
    { role: "system", content: "你是一个前端学习助手。" },
    { role: "user", content: "解释 JavaScript 数据类型。" }
  ]
};
```
