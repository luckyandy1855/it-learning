# 前端从入门到 AI 应用开发学习路线图

## 🎯 学习目标

这套课程的目标不是只学会写页面，而是逐步具备从 0 到 1 构建 AI Web 应用的能力：能写结构、能做样式、能实现交互、理解浏览器运行机制、能用 React 和 TypeScript 组织复杂界面，并最终接入 AI API、RAG、MCP 和完整工程化交付流程。

适合的学习对象：

- 想从前端基础进入 AI 应用开发的人。
- 已会一点 HTML/CSS/JS，但缺少系统路线的人。
- 想做 AI Chat、Prompt 工具、知识库助手、Agent 前端或 MCP 工具界面的人。

---

## 🧭 总体路线

| 阶段 | 模块 | 核心目标 | 产出能力 |
|------|------|----------|----------|
| 0 | HTML | 建立页面语义和结构能力 | 能写 AI 工具页面骨架 |
| 1 | CSS | 建立布局、响应式和视觉状态能力 | 能做聊天界面、仪表盘和主题系统 |
| 2 | JavaScript | 建立交互、DOM、异步和流式基础 | 能实现 Prompt 输入、消息渲染和接口调用 |
| 3 | 浏览器工作原理 | 理解网络、渲染、缓存和性能 | 能定位 AI 页面慢、卡顿和加载问题 |
| 4 | React | 用组件和状态组织复杂 AI 界面 | 能构建完整 AI Chat 前端 |
| 5 | TypeScript | 为消息、请求、响应和组件建立类型契约 | 能降低 AI 应用重构和接口变更风险 |
| 6 | 前端工程化 | 建立开发、检查、测试、构建和部署流程 | 能交付可维护项目 |

## 📚 推荐学习顺序

1. `01-HTML`

   先掌握页面结构、语义标签、表单、媒体、SEO 和无障碍。AI Web 的聊天窗口、Prompt 表单、知识库页面都离不开 HTML 结构。

2. `02-CSS`

   学布局、响应式、动画、主题和 Tailwind。重点关注 AI 聊天气泡、消息列表、仪表盘、骨架屏、Typing 光标和暗色模式。

3. `03-JavaScript`

   学变量、函数、数组、对象、DOM、事件、fetch、Promise、async/await、Event Loop、SSE/WebSocket 和模块化。这是 AI Web 交互与接口调用的基础。

4. `04-浏览器工作原理`

   理解 URL 加载、DNS、HTTPS、HTTP/2、DOM/CSSOM、Layout、Paint、Event Loop、缓存、性能和 GC。遇到 AI 应用慢、卡顿、加载失败时，需要靠这些知识定位问题。

5. `05-React`

   用组件、Props、State、Effect、Context、Router、Zustand、TanStack Query、自定义 Hook 和测试组织复杂 AI Chat 界面。

6. `06-TypeScript`

   为消息、模型配置、请求体、响应体、流式事件、工具调用和 React 组件建立类型。AI 应用的数据结构复杂，类型系统能显著降低维护成本。

7. `07-前端工程化`

   建立 Vite、ESLint、Prettier、测试、CI、构建、部署、Docker 和质量门禁。目标是让项目可协作、可检查、可部署。

8. `08-AI-Web开发`

   学 OpenAI/Anthropic SDK、后端代理、Streaming、Embedding、RAG、Prompt 模板、工具调用、内容安全、成本控制和可观测性。

9. `09-MCP开发`

   学 MCP Server、Tool、Resource、Prompt、Client 接入、调试、权限和审计。目标是让 AI Agent 能安全调用外部能力。

10. `10-综合项目实战`

    把前面所有模块串起来，完成一个 AI 知识库助手，从需求定义、架构设计、工程初始化、AI Chat、RAG、MCP、测试到部署复盘。

| 周期 | 学习内容 | 建议产出 |

| 第 11 周 | MCP | 自定义 MCP Tool/Resource 小服务 |

## 📌 学习方法

- 每篇文章先看「学习目标」和「完整代码」，再回头看原理。
- 每学完 5 篇，至少改写一个示例，不要只阅读。
- 所有涉及 API Key 的示例，都默认放在后端，不放在浏览器前端。
- 学到 React 后，把前面 HTML/CSS/JS 的静态示例逐步组件化。
- 学到 AI Web 后，把假数据回复替换为后端代理和流式输出。
- 学到 MCP 后，为综合项目增加一个真实工具能力，例如读取课程目录或生成复盘建议。

## ✅ 最终目标

完成本路线后，你应该能独立完成：

- 一个可用的 AI Chat 前端。
- 一个安全的后端代理接口设计。
- 一个基础 RAG 知识库问答流程。
- 一个可接入 AI Agent 的 MCP Server。
- 一套可检查、可测试、可构建、可部署的前端工程流程。
- 一份项目复盘和长期维护清单。
