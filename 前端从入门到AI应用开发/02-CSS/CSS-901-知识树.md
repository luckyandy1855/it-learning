# CSS 知识树（CSS-901）

## 🎯 本节学习目标

- 建立 CSS 模块从基础语法到 AI 界面实战的完整知识地图。
- 理解选择器、盒模型、布局、响应式、动画、主题和 Tailwind 的学习顺序。
- 能够按知识树复盘 AI Chat、AI 仪表盘和 Prompt 工具页面的样式能力。

---

## 📖 什么是 CSS 知识树

CSS 知识树是对 CSS-001 到 CSS-060 的结构化复盘。它不是简单目录，而是把知识点按能力层级组织起来，帮助学习者知道“先学什么、后学什么、每块知识解决什么问题”。

在 AI 应用开发中，CSS 的价值不只是美化页面。它决定消息是否易读、状态是否清晰、布局是否稳定、移动端是否可用、长文本和代码块是否能被正确承载。

---

## 🧠 原理讲解

CSS 学习可以分成六层：

```text
选择器与层叠
  ↓
盒模型与显示类型
  ↓
Flex / Grid / Position
  ↓
响应式与容器适配
  ↓
动画、层级、溢出、滤镜
  ↓
主题系统与 Tailwind 工程化写法
```

越靠前越基础，越靠后越接近真实 AI 产品界面。

## ✅ 完整代码

下面是一个覆盖 CSS 核心能力的 AI Chat 面板样式：

```html
<main class="chat-shell" data-theme="light">
  <section class="message-list">
    <article class="message user">请总结这段内容。</article>
    <article class="message assistant">可以，我会提取重点并给出行动项。</article>
  </section>
  <form class="prompt-bar">
    <textarea placeholder="输入 Prompt"></textarea>
    <button type="button">发送</button>
  </form>
</main>

<style>
  :root {
    --bg: #f8fafc;
    --panel: #ffffff;
    --text: #111827;
    --border: #d1d5db;
    --accent: #2563eb;
  }

  .chat-shell {
    display: grid;
    gap: 16px;
    max-width: min(900px, 92vw);
    margin: 32px auto;
    color: var(--text);
  }

  .message-list {
    display: flex;
    flex-direction: column;
    gap: 12px;
  }

  .message {
    max-width: 76%;
    padding: 14px 16px;
    border: 1px solid var(--border);
    border-radius: 8px;
    line-height: 1.7;
    background: var(--panel);
  }

  .message.user {
    align-self: flex-end;
    border-color: var(--accent);
  }

  .prompt-bar {
    display: flex;
    gap: 12px;
    position: sticky;
    bottom: 16px;
  }

  .prompt-bar textarea {
    flex: 1;
    min-width: 0;
    padding: 12px;
  }

  @media (max-width: 640px) {
    .message {
      max-width: 100%;
    }

    .prompt-bar {
      flex-direction: column;
    }
  }
</style>
```

---

## 🔍 逐行解析

`:root` 定义主题变量，后续颜色都从语义变量读取。

`.chat-shell` 使用 Grid 管理整体垂直节奏，并通过 `max-width` 控制阅读宽度。

`.message-list` 使用 Flex 纵向排列消息。

`.message` 定义消息气泡的边框、内边距、圆角和行高。

`.message.user` 通过 `align-self: flex-end` 实现用户消息靠右。

`.prompt-bar` 使用 sticky，让输入栏在滚动时保持可访问。

媒体查询让移动端消息占满宽度，输入栏改为纵向排列。

## 🌐 浏览器表现

浏览器会渲染一个居中的 AI 聊天面板。消息按上下排列，用户消息靠右，AI 消息靠左，底部输入栏在滚动时保持可见。

在 DevTools 中可以分别检查 Flex 对齐、CSS 变量、媒体查询和 sticky 定位是否生效。

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `display` | CSS 属性 | 切换布局模式 | `display: flex;` |
| `gap` | CSS 属性 | 控制项目间距 | `gap: 12px;` |
| `max-width` | CSS 属性 | 控制阅读宽度 | `max-width: 900px;` |
| `position` | CSS 属性 | 控制定位方式 | `position: sticky;` |
| `var()` | CSS 函数 | 读取 CSS 变量 | `color: var(--text);` |
| `@media` | CSS 规则 | 响应式适配 | `@media (max-width: 640px)` |

## ⭐⭐⭐⭐⭐ 必学重点

- CSS 能力要按“选择器 -> 盒模型 -> 布局 -> 响应式 -> 状态 -> 主题”逐层建立。
- AI 界面最常用的是 Flex、Grid、Position、overflow、动画和 CSS 变量。
- 可读性、状态反馈和移动端输入体验，比装饰性效果更重要。

---

## ⚠️ 易错点

- 易错点1：只学属性，不知道应用场景。正确做法：每个属性都放到具体 AI 界面里练习。
- 易错点2：用 fixed、absolute 硬做所有布局。正确做法：普通布局优先 Flex/Grid，浮层再用定位。
- 易错点3：忽略长文本和长代码。正确做法：为消息正文和代码块设置 `min-width: 0`、`overflow` 和合理行高。

---

## 💡 最佳实践

- 先定义全局字体、颜色、间距和容器宽度。
- 复杂页面先用 Grid 定骨架，再用 Flex 做局部对齐。
- AI 生成中、错误、禁用、引用来源等状态都要有明确样式。
- 主题系统优先使用语义变量，而不是到处写具体颜色。

## 🚀 AI 应用场景

CSS 知识树可以直接映射到 AI 产品界面：

```css
.ai-layout {
  display: grid;
  grid-template-columns: 260px minmax(0, 1fr) 300px;
  gap: 16px;
}

.ai-answer {
  min-width: 0;
  overflow-wrap: anywhere;
  line-height: 1.8;
}
```

这类写法适合知识库侧栏、主对话区和参数面板组成的 AI 工作台。
