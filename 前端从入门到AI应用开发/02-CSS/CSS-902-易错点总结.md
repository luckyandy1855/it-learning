# CSS 易错点总结（CSS-902）

## 🎯 本节学习目标

- 复盘 CSS 模块中最常见的布局、优先级、响应式和状态样式错误。
- 掌握从 DevTools 反查样式问题的基本思路。
- 能够排查 AI Chat、AI 仪表盘和 Prompt 工具页面中的高频 CSS 问题。

---

## 📖 什么是 CSS 易错点总结

CSS 易错点总结用于把零散踩坑沉淀成可复用检查清单。CSS 很多问题不是属性不会写，而是写在了错误位置、被其他规则覆盖、没有考虑内容长度或没有覆盖交互状态。

AI 应用里，这些问题更明显：模型回复很长、代码块很宽、输入栏固定、消息流式追加、弹窗层级复杂、移动端输入困难。

---

## 🧠 原理讲解

排查 CSS 问题可以按这条链路：

```text
元素是否选中
  ↓
规则是否命中
  ↓
属性是否被覆盖
  ↓
布局上下文是否正确
  ↓
内容是否超出
  ↓
响应式和状态是否覆盖
```

不要只看代码，要用 DevTools 验证最终生效的 computed style。

## 🏗 基本结构

```css
.message-body {
  flex: 1;
  min-width: 0;
  overflow-wrap: anywhere;
}
```

## ✅ 完整代码

下面是一组常见错误和修正写法：

```html
<div class="message-row">
  <aside class="avatar">AI</aside>
  <article class="message-body">
    这是一段很长的 AI 回复，可能包含很长的链接、代码或连续文本。
  </article>
</div>

<style>
  .message-row {
    display: flex;
    gap: 12px;
    max-width: 720px;
  }

  .avatar {
    flex: 0 0 40px;
    height: 40px;
    border-radius: 50%;
    background: #dbeafe;
    text-align: center;
    line-height: 40px;
  }

  .message-body {
    flex: 1;
    min-width: 0;
    padding: 12px 14px;
    border: 1px solid #e5e7eb;
    border-radius: 8px;
    overflow-wrap: anywhere;
    line-height: 1.7;
  }

  .message-body:focus-visible {
    outline: 3px solid #bfdbfe;
    outline-offset: 2px;
  }
</style>
```

---

## 🔍 逐行解析

`.message-row` 建立 Flex 布局，头像和正文横向排列。

`.avatar` 用固定 flex basis 保持头像宽度稳定。

`.message-body` 使用 `flex: 1` 占据剩余空间。

`min-width: 0` 防止长文本撑破 Flex 容器。

`overflow-wrap: anywhere` 允许超长链接或连续文本换行。

`:focus-visible` 保留键盘可访问的焦点状态。

---

## 🌐 浏览器表现

浏览器会显示一个头像和消息正文并排的 AI 消息行。即使正文包含长文本，也会在容器内换行，不会把页面撑出横向滚动。

在 DevTools 中可以删除 `min-width: 0` 或 `overflow-wrap` 观察布局被撑破的效果。

---

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `min-width` | CSS 属性 | 控制弹性项目最小宽度 | `min-width: 0;` |
| `overflow-wrap` | CSS 属性 | 控制长文本换行 | `overflow-wrap: anywhere;` |
| `box-sizing` | CSS 属性 | 控制盒模型计算方式 | `border-box` |
| `z-index` | CSS 属性 | 控制层级 | `z-index: 100;` |
| `focus-visible` | 伪类 | 键盘焦点样式 | `:focus-visible` |
| DevTools Computed | 调试面板 | 查看最终生效样式 | Computed 面板 |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 样式不生效时，先检查选择器、优先级和覆盖关系。
- Flex 子项长文本撑破布局时，优先检查 `min-width: 0`。
- AI 页面必须覆盖 hover、focus、disabled、loading、error 和 streaming 状态。

---

## ⚠️ 易错点

- 易错点1：滥用 ID 和 `!important`。正确写法：优先用类名和清晰层级管理样式。
- 易错点2：移除 outline 后没有替代焦点样式。正确写法：使用 `:focus-visible` 提供可见焦点。
- 易错点3：弹窗层级低于固定输入栏。正确写法：统一设计 z-index 层级表。
- 易错点4：移动端仍使用桌面三栏布局。正确写法：窄屏切换为单列。

---

## 💡 最佳实践

- 为布局、组件、状态、主题分别建立命名规则。
- 调试时先看 Elements 和 Computed，再改代码。
- 为长文本、长代码、固定输入栏和弹窗写专门测试页面。
- 对 Tailwind 工具类组合进行抽象，避免一行 class 过长且不可维护。

---

## 🚀 AI 应用场景

AI 回复经常包含长 Markdown、代码块和链接。下面是防止内容撑破布局的关键样式：

```css
.ai-answer {
  min-width: 0;
  overflow-wrap: anywhere;
}

.ai-answer pre {
  max-width: 100%;
  overflow-x: auto;
}
```
