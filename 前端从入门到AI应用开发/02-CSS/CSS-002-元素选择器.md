# 元素选择器（CSS-002）

## 🎯 本节学习目标

- 掌握元素选择器的写法和作用范围。
- 理解为什么元素选择器适合定义基础排版规则。
- 能够用元素选择器为 AI 文档页建立统一阅读样式。

---

## 📖 什么是元素选择器

元素选择器也叫标签选择器，它直接使用 HTML 标签名作为选择器，例如 `body`、`h1`、`p`、`button`。

它的作用是选中页面中所有对应标签。比如 `p { line-height: 1.7; }` 会影响当前页面里的每一个段落。

元素选择器适合写基础样式，例如正文行高、标题间距、按钮默认字体，而不适合表达复杂业务含义。

---

## 🧠 原理讲解

浏览器匹配 CSS 时，会检查页面中的元素是否符合选择器条件。元素选择器的条件很简单：标签名相同就匹配。

```css
h1 {
  font-size: 32px;
}
```

这段规则会匹配所有 `<h1>`。如果页面有三个一级标题，三个都会变成 32px。

因为影响范围较广，元素选择器通常用于“全站基础规则”，而不是用于某个具体组件的细节样式。

---

## 🏗 基本结构

```html
<style>
  h1 {
    font-size: 32px;
  }

  p {
    line-height: 1.7;
  }
</style>

<h1>AI 笔记</h1>
<p>这是一段由 AI 生成的学习总结。</p>
```

---

## ✅ 完整代码

下面用元素选择器创建一个 AI 摘要页面的基础排版：

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>元素选择器示例</title>
  <style>
    body {
      margin: 0;
      padding: 32px;
      font-family: system-ui, sans-serif;
      background: #f8fafc;
      color: #111827;
    }

    main {
      max-width: 760px;
      margin: 0 auto;
      background: #ffffff;
      padding: 28px;
      border: 1px solid #e5e7eb;
      border-radius: 8px;
    }

    h1 {
      margin: 0 0 16px;
      font-size: 30px;
    }

    h2 {
      margin-top: 28px;
      font-size: 20px;
    }

    p {
      margin: 0 0 14px;
      line-height: 1.8;
    }

    button {
      padding: 10px 14px;
      border: 1px solid #2563eb;
      border-radius: 6px;
      background: #2563eb;
      color: #ffffff;
      font: inherit;
    }
  </style>
</head>
<body>
  <main>
    <h1>AI 文章摘要</h1>
    <p>本页面展示模型对长文的摘要结果。</p>

    <h2>核心观点</h2>
    <p>CSS 的选择器决定样式应用到哪些元素。</p>

    <button>重新生成摘要</button>
  </main>
</body>
</html>
```

---

## 🔍 逐行解析

`body` 规则影响整个页面，负责设置基础字体、背景和文字颜色。

`main` 规则影响主内容区域，让正文宽度适中，避免长行影响阅读。

`h1` 和 `h2` 分别设置标题大小和间距，让页面层次更清楚。

`p` 统一控制段落间距和行高，适合 AI 输出的长文本阅读。

`button` 统一按钮默认视觉，`font: inherit;` 让按钮继承页面字体。

---

## 🌐 浏览器表现

所有 `<p>` 都会自动获得相同的行高和底部间距。新增段落时，不需要额外写类名也能保持一致排版。

如果在 DevTools 中临时关闭 `p` 规则，可以看到段落变得更拥挤，这能直观说明元素选择器对基础排版的价值。

---

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `body` | 元素选择器 | 选择页面主体 | `body { margin: 0; }` |
| `h1` | 元素选择器 | 选择一级标题 | `h1 { font-size: 30px; }` |
| `p` | 元素选择器 | 选择所有段落 | `p { line-height: 1.8; }` |
| `button` | 元素选择器 | 选择所有按钮 | `button { font: inherit; }` |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 元素选择器用标签名匹配元素。
- 元素选择器影响范围广，适合基础排版。
- 不要用元素选择器承担复杂组件状态样式。

---

## ⚠️ 易错点

- 易错点1：给 `div` 写大量全局样式。正确写法：`div` 太通用，组件样式优先使用类选择器。
- 易错点2：用 `button` 规则覆盖所有按钮细节。正确写法：元素选择器只写基础样式，变体交给类名。
- 易错点3：忘记元素选择器会影响新增内容。正确写法：写全局规则前先判断影响范围。

---

## 💡 最佳实践

- 用元素选择器设置 `body`、标题、段落、表单控件的基础风格。
- 组件内部细节优先使用类选择器，避免误伤全局页面。
- 在 AI 长文本页面中，优先为 `p`、`ul`、`ol`、`pre` 建立阅读规则。

---

## 🚀 AI 应用场景

AI 会生成大量结构化内容，例如标题、段落、列表和代码块。元素选择器可以为这些 Markdown 渲染后的 HTML 节点提供统一基础样式。

```css
article h2 {
  margin-top: 32px;
}

article p {
  line-height: 1.85;
}

article code {
  background: #f3f4f6;
  padding: 2px 4px;
  border-radius: 4px;
}
```
