# 伪类选择器（CSS-006）

## 🎯 本节学习目标

- 掌握伪类选择器的基本写法，例如 `:hover`、`:focus`、`:disabled`。
- 理解伪类用于描述元素状态，而不是新增 HTML 结构。
- 能够为 AI 应用中的按钮、输入框和列表项设计清晰交互状态。

---

## 📖 什么是伪类选择器

伪类选择器用于选择处于特定状态的元素。它不是 HTML 里真实存在的类名，而是浏览器根据用户操作、元素位置或表单状态动态判断出来的状态。

常见伪类包括 `:hover`、`:focus`、`:active`、`:disabled`、`:checked`、`:first-child`、`:last-child`。

在 AI 应用中，伪类常用于按钮 hover 效果、Prompt 输入框聚焦状态、禁用的发送按钮、消息列表第一项和最后一项的间距处理。

---

## 🧠 原理讲解

伪类写在选择器后面，用英文冒号连接：

```css
button:hover {
  background: #1d4ed8;
}
```

这条规则表示：当鼠标悬停在 `<button>` 上时，按钮背景色改变。

浏览器会根据当前交互状态实时判断伪类是否匹配。状态变化时，浏览器重新计算相关元素样式，不需要 JavaScript 手动添加类名。

---

## 🏗 基本结构

```html
<style>
  button:hover {
    background: #1d4ed8;
  }

  textarea:focus {
    border-color: #2563eb;
  }
</style>

<textarea placeholder="输入 Prompt"></textarea>
<button>发送</button>
```

---

## 🔍 逐行解析

`.prompt-input:focus` 表示文本框获得焦点时应用边框和外轮廓样式。

`.send-button:hover` 表示鼠标悬停在按钮上时加深背景色，让按钮有可点击反馈。

`.send-button:disabled` 表示按钮处于禁用状态时变灰，并把鼠标指针改成不可操作样式。

这些状态都不需要额外写 JavaScript，浏览器会自动根据用户操作和元素属性判断。

---

## 🌐 浏览器表现

用户点击文本框时，边框会变蓝并显示浅蓝外轮廓。鼠标悬停在普通按钮上时，按钮颜色会变深。禁用按钮保持灰色，无法点击。

在 DevTools 的 Styles 面板中，可以手动勾选 `:hover`、`:focus` 等状态来调试伪类效果。

---

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `:hover` | 伪类 | 鼠标悬停状态 | `.button:hover` |
| `:focus` | 伪类 | 元素获得焦点 | `input:focus` |
| `:disabled` | 伪类 | 表单控件禁用状态 | `button:disabled` |
| `:first-child` | 伪类 | 父元素中的第一个子元素 | `li:first-child` |
| `:last-child` | 伪类 | 父元素中的最后一个子元素 | `li:last-child` |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 伪类描述元素状态，不需要 HTML 中真的写 `class="hover"`。
- 交互状态优先用伪类处理，减少不必要的 JavaScript。
- `:focus` 对键盘用户和无障碍体验非常重要。

---

## ⚠️ 易错点

- 易错点1：只写 `:hover`，不写 `:focus`。正确写法：键盘可操作元素也要有清晰焦点状态。
- 易错点2：禁用按钮只改颜色。正确写法：同时设置 `cursor: not-allowed` 并确保按钮真的有 `disabled` 属性。
- 易错点3：把伪类当成 HTML 类名。正确写法：CSS 写 `button:hover`，HTML 不需要写 `hover`。

---

## 💡 最佳实践

- 所有可点击元素都应该有 hover 或 active 反馈。
- 所有输入控件都应该有清晰的 focus 样式。
- AI 请求处理中，应使用 `:disabled` 显示按钮不可操作状态。

---

## 🚀 AI 应用场景

AI 应用常见“发送中”“生成中”“重新生成”等状态。伪类可以让这些控件在不用额外状态类的情况下具备基本交互反馈。

```css
.model-option:hover {
  border-color: #2563eb;
}

.model-option:focus-within {
  outline: 3px solid #bfdbfe;
}

.generate-button:disabled {
  opacity: 0.6;
}
```
