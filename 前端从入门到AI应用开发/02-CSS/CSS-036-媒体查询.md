# 媒体查询（CSS-036）

## 🎯 本节学习目标

- 掌握 `@media` 媒体查询的基本写法。
- 理解断点用于在不同屏幕宽度下调整布局。
- 能够为移动端 AI 聊天界面编写基础响应式样式。

---

## 📖 什么是媒体查询

媒体查询是 CSS 响应式设计的核心能力。它允许我们根据屏幕宽度、设备类型、用户偏好等条件应用不同样式。

最常见的使用方式是根据视口宽度设置断点，例如大屏显示两列，小屏改成一列。

在 AI 应用中，聊天界面、仪表盘、侧边栏和输入栏都需要响应式处理，否则在手机上容易出现横向滚动、文字拥挤或按钮被遮挡。

---

## 🧠 原理讲解

基础写法：

```css
.layout {
  display: grid;
  grid-template-columns: 240px 1fr;
}

@media (max-width: 768px) {
  .layout {
    grid-template-columns: 1fr;
  }
}
```

这表示：默认使用两列布局；当视口宽度小于等于 768px 时，改为一列布局。

响应式设计的关键不是写很多断点，而是在内容真正放不下时调整布局。

---

## 🏗 基本结构

```html
<style>
  .cards {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
  }

  @media (max-width: 720px) {
    .cards {
      grid-template-columns: 1fr;
    }
  }
</style>
```

## ✅ 完整代码

下面为 AI 控制台编写移动端适配：

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>媒体查询示例</title>
  <style>
    * {
      box-sizing: border-box;
    }

    body {
      margin: 0;
      padding: 32px;
      font-family: system-ui, sans-serif;
      background: #f8fafc;
      color: #111827;
    }

    .ai-layout {
      display: grid;
      grid-template-columns: 240px minmax(0, 1fr);
      gap: 16px;
      max-width: 1080px;
      margin: 0 auto;
    }

    .panel {
      padding: 20px;
      border: 1px solid #e5e7eb;
      border-radius: 8px;
      background: #ffffff;
    }

    .message {
      line-height: 1.8;
    }

    @media (max-width: 768px) {
      body {
        padding: 16px;
      }

      .ai-layout {
        grid-template-columns: 1fr;
      }

      .panel {
        padding: 16px;
      }
    }
  </style>
</head>
<body>
  <main class="ai-layout">
    <aside class="panel">知识库、模板和模型设置</aside>
    <section class="panel">
      <h1>AI 对话</h1>
      <p class="message">小屏幕下，侧边栏会移动到主内容上方，避免横向拥挤。</p>
    </section>
  </main>
</body>
</html>
```

---

## 🔍 逐行解析

`.ai-layout` 默认使用两列：左侧固定 240px，右侧占剩余空间。

`@media (max-width: 768px)` 表示当屏幕较窄时应用内部样式。

小屏下 `grid-template-columns: 1fr` 把两列改成一列。

`body` 和 `.panel` 的 padding 也同步缩小，避免手机上空间浪费。

---

## 🌐 浏览器表现

桌面端会显示左侧栏和主内容两列。缩小浏览器宽度到 768px 以下时，布局变成单列，内容更适合手机阅读。

在 DevTools 中切换设备模式，可以观察媒体查询在不同宽度下是否生效。

---

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `@media` | CSS 规则 | 定义媒体查询 | `@media (max-width: 768px)` |
| `max-width` | 媒体特性 | 小于等于指定宽度时生效 | `(max-width: 768px)` |
| `min-width` | 媒体特性 | 大于等于指定宽度时生效 | `(min-width: 1024px)` |
| 断点 | 设计概念 | 样式切换的宽度 | `768px` |
| DevTools 设备模式 | 调试工具 | 模拟不同屏幕 | 浏览器开发者工具 |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 媒体查询用于根据条件应用不同 CSS。
- 响应式断点应由内容需要决定。
- 小屏常见处理是多列变单列、间距缩小、固定侧栏取消。

---

## ⚠️ 易错点

- 易错点1：只改宽度，不改间距。正确写法：小屏布局、padding、字号都要整体检查。
- 易错点2：断点写太多导致维护困难。正确写法：从关键布局断点开始。
- 易错点3：忘记测试真实长内容。正确写法：用长 Prompt、长回答和长模型名测试。

---

## 💡 最佳实践

- 优先写默认布局，再用媒体查询处理必要断点。
- 常见断点可以从 768px、1024px 起步，但最终以内容表现为准。
- AI 聊天界面小屏优先保证阅读和输入，不强行保留桌面侧栏。

---

## 🚀 AI 应用场景

移动端 AI 助手通常需要把桌面两列布局降级为单列，并减少边距，保证输入框和回答内容可用。

```css
@media (max-width: 768px) {
  .ai-console {
    grid-template-columns: 1fr;
  }

  .side-panel {
    order: -1;
  }
}
```
