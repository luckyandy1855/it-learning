# CSS 综合练习题（CSS-903）

## 🎯 本节学习目标

- 通过综合练习复盘 CSS 模块的核心知识。
- 能够把选择器、布局、响应式、动画、主题和 Tailwind 串成实际页面。
- 能够独立完成一个 AI Chat 或 AI 控制台的样式方案。

---

## 📖 什么是 CSS 综合练习题

CSS 综合练习题用于检验你是否真的能把知识点用到页面里。单独知道某个属性不够，真实项目需要同时处理布局、状态、响应式、可访问性和维护性。

本节练习围绕 AI 应用展开，包括聊天页面、控制台仪表盘、生成报告页和移动端 Prompt 工具。

---

## 🧠 原理讲解

综合练习可以按四步完成：

```text
拆结构
  ↓
定布局
  ↓
补状态
  ↓
做响应式和自查
```

先用 HTML 定义清晰区域，再用 CSS 控制页面骨架、组件样式、状态反馈和移动端适配。

## ✅ 完整代码

下面是综合练习的起始模板：

```html
<main class="ai-page">
  <aside class="sidebar">知识库</aside>
  <section class="chat-panel">
    <article class="message user">请生成一份复盘。</article>
    <article class="message assistant">可以，我会按背景、过程、结论整理。</article>
  </section>
  <aside class="settings">模型参数</aside>
  <form class="prompt-bar">
    <textarea placeholder="输入 Prompt"></textarea>
    <button type="button">发送</button>
  </form>
</main>

<style>
  .ai-page {
    display: grid;
    grid-template-columns: 240px minmax(0, 1fr) 280px;
    gap: 16px;
    padding: 16px;
  }

  .chat-panel {
    display: flex;
    flex-direction: column;
    gap: 12px;
    min-width: 0;
  }

  .message {
    padding: 14px 16px;
    border: 1px solid #e5e7eb;
    border-radius: 8px;
    line-height: 1.7;
    overflow-wrap: anywhere;
  }

  .message.user {
    align-self: flex-end;
  }

  .settings {
    position: sticky;
    top: 16px;
    align-self: start;
  }

  .prompt-bar {
    grid-column: 2;
    display: flex;
    gap: 12px;
    position: sticky;
    bottom: 16px;
  }

  .prompt-bar textarea {
    flex: 1;
    min-width: 0;
  }

  @media (max-width: 900px) {
    .ai-page {
      grid-template-columns: 1fr;
    }

    .prompt-bar {
      grid-column: 1;
      flex-direction: column;
    }
  }
</style>
```

---

## 🔍 逐行解析

`.ai-page` 使用 Grid 创建三栏 AI 控制台。

`minmax(0, 1fr)` 防止主内容区域被长文本撑破。

`.chat-panel` 使用 Flex 纵向排列消息。

`.message` 定义消息气泡的基础样式。

`.settings` 使用 sticky，桌面端滚动时保持参数面板可见。

`.prompt-bar` 固定在主内容底部附近，便于持续输入。

媒体查询在窄屏下切换为单列布局。

---

## 🌐 浏览器表现

桌面端会显示左侧知识库、中间聊天区、右侧参数面板。移动端会变成单列布局，输入栏改为纵向排列。

在 DevTools 中可以拖动视口宽度，观察 Grid 断点、sticky 面板和输入栏布局是否稳定。

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `grid-template-columns` | CSS 属性 | 定义页面列结构 | `240px 1fr 280px` |
| `minmax()` | CSS 函数 | 定义弹性范围 | `minmax(0, 1fr)` |
| `position: sticky` | CSS 属性 | 滚动粘性定位 | `top: 16px` |
| `overflow-wrap` | CSS 属性 | 处理长文本 | `anywhere` |
| `@media` | CSS 规则 | 响应式断点 | `max-width: 900px` |
| `flex-direction` | CSS 属性 | 控制 Flex 主轴方向 | `column` |

## ⭐⭐⭐⭐⭐ 必学重点

- 综合练习要同时覆盖结构、布局、状态和响应式。
- AI 页面重点处理长文本、代码块、固定输入栏、弹窗层级和生成中状态。
- 完成页面后必须用 DevTools 检查桌面和移动端。

---

## ⚠️ 易错点

- 易错点1：只完成桌面布局。正确做法：必须写移动端断点。
- 易错点2：忽略长代码块。正确做法：为 `pre` 设置 `overflow-x: auto`。
- 易错点3：没有 focus 和 disabled 状态。正确做法：交互控件至少覆盖 hover、focus-visible、disabled。

---

## 💡 最佳实践

- 先写页面骨架，再写组件样式，最后补状态和响应式。
- 每道题完成后检查横向滚动、层级、焦点、禁用、加载和错误状态。
- 用语义清晰的类名，避免为了短期效果写不可维护选择器。
