# 容器查询（CSS-040）

## 🎯 本节学习目标

- 理解容器查询根据组件容器尺寸应用样式。
- 掌握 `container-type` 和 `@container` 的基础写法。
- 能够让 AI 卡片组件在不同容器中自适应，而不只依赖视口宽度。

---

## 📖 什么是容器查询

媒体查询关注的是视口尺寸，容器查询关注的是组件所在容器的尺寸。

同一个组件可能出现在主内容区、侧边栏、弹窗或仪表盘卡片中。视口宽度相同，但组件可用空间不同，这时容器查询比媒体查询更精确。

在 AI 应用中，模型卡片、Prompt 模板卡片、引用来源卡片经常复用到不同区域，适合使用容器查询做组件级响应式。

---

## 🧠 原理讲解

先把父容器声明为查询容器：

```css
.card-host {
  container-type: inline-size;
}
```

再写容器查询：

```css
@container (max-width: 360px) {
  .model-card {
    grid-template-columns: 1fr;
  }
}
```

这表示当查询容器的行内尺寸小于等于 360px 时，内部卡片改变布局。

---

## 🏗 基本结构

```html
<style>
  .host {
    container-type: inline-size;
  }

  @container (max-width: 360px) {
    .card {
      display: block;
    }
  }
</style>

<div class="host">
  <article class="card">AI 模型卡片</article>
</div>
```

## 🔍 逐行解析

`.card-host { container-type: inline-size; }` 把每个卡片所在区域声明为查询容器。

`.model-card` 默认使用三列布局：图标、内容、按钮。

`@container (max-width: 360px)` 根据容器宽度触发，而不是根据浏览器宽度触发。

在窄侧边栏中，卡片会变成单列布局，按钮宽度变为 100%。

---

## 🌐 浏览器表现

主内容区里的模型卡片可能保持横向布局，侧边栏里的同款卡片因为容器更窄，会自动变成纵向布局。

这说明容器查询可以让同一个组件在不同容器中表现不同，不必为每个页面单独写样式。

---

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `container-type` | CSS 属性 | 声明查询容器类型 | `inline-size` |
| `@container` | CSS 规则 | 根据容器尺寸应用样式 | `@container (max-width: 360px)` |
| `inline-size` | CSS 值 | 查询行内方向尺寸 | 常用于宽度 |
| 容器查询 | 响应式技术 | 组件按容器适配 | 卡片复用 |
| 媒体查询 | 对比技术 | 页面按视口适配 | `@media` |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 媒体查询看视口，容器查询看组件容器。
- 使用容器查询前，要先声明查询容器。
- 复用组件在不同区域宽度不同，适合容器查询。

---

## ⚠️ 易错点

- 易错点1：写了 `@container` 却没有声明容器。正确写法：父级加 `container-type: inline-size`。
- 易错点2：把页面级布局全改成容器查询。正确写法：页面骨架仍常用媒体查询，组件内部适配用容器查询。
- 易错点3：选择了错误容器。正确写法：查询容器应是组件实际受限的外层。

---

## 💡 最佳实践

- 组件库、卡片、列表项适合容器查询。
- 页面整体断点用媒体查询，组件局部断点用容器查询。
- 容器查询断点应根据组件内容是否拥挤决定。

---

## 🚀 AI 应用场景

AI 模型卡片可能同时出现在模型市场、侧边参数栏和弹窗里。容器查询能让同一个组件根据可用空间自动调整。

```css
.model-card-host {
  container-type: inline-size;
}

@container (max-width: 360px) {
  .model-card {
    grid-template-columns: 1fr;
  }
}
```

3. [AI 场景题] 设计一个 AI 模型卡片，让它在主内容区横排，在侧边栏竖排。
