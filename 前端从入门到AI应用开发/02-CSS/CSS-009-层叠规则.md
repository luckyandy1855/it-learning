# 层叠规则（CSS-009）

## 🎯 本节学习目标

- 理解 CSS 中“层叠”的含义。
- 掌握来源、优先级、书写顺序对最终样式的影响。
- 能够排查 AI 界面中样式没有生效或被覆盖的问题。

---

## 📖 什么是层叠规则

CSS 的 Cascading 指的就是“层叠”。当多个规则同时命中同一个元素，并且设置了同一个属性时，浏览器需要决定最终使用哪一个值。

这个决定过程不是随机的，而是按照一套规则比较：样式来源、重要性、选择器优先级、代码书写顺序。

理解层叠规则，是排查 CSS 问题的基础。很多“为什么颜色没变”“为什么按钮样式不生效”，本质都是层叠规则导致的。

---

## 🧠 原理讲解

当同一个元素被多条规则命中时，浏览器大致按以下顺序判断：

1. `!important` 是否存在。
2. 选择器优先级谁更高。
3. 如果优先级相同，谁写在后面。

示例：

```css
.button {
  background: gray;
}

.primary {
  background: blue;
}
```

如果元素是 `<button class="button primary">`，两条规则优先级相同，后写的 `.primary` 生效。

---

## 🏗 基本结构

```html
<style>
  .message {
    color: #111827;
  }

  .message {
    color: #2563eb;
  }
</style>

<p class="message">最终显示蓝色。</p>
```

## ✅ 完整代码

下面演示 AI 消息样式如何被后续规则覆盖：

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>层叠规则示例</title>
  <style>
    body {
      margin: 0;
      padding: 32px;
      font-family: system-ui, sans-serif;
      background: #f8fafc;
      color: #111827;
    }

    .message {
      padding: 12px 14px;
      border-radius: 8px;
      background: #f3f4f6;
      color: #111827;
      line-height: 1.7;
    }

    .message-ai {
      background: #ffffff;
      border: 1px solid #e5e7eb;
    }

    .message-important {
      background: #fef3c7;
      color: #92400e;
    }

    .message-ai {
      color: #1f2937;
    }
  </style>
</head>
<body>
  <p class="message message-ai">普通 AI 回复：使用 AI 消息样式。</p>
  <p class="message message-ai message-important">重要 AI 回复：背景和文字被重要样式覆盖。</p>
</body>
</html>
```

---

## 🔍 逐行解析

`.message` 提供所有消息共有的基础样式。

`.message-ai` 第一次出现时设置 AI 回复的背景和边框。

`.message-important` 给重要消息设置黄色背景和深色文字。

最后一条 `.message-ai` 再次设置文字颜色。因为它写在后面，并且与前面的类选择器优先级相同，所以会覆盖同优先级的文字颜色。

---

## 🌐 浏览器表现

普通 AI 回复显示为白底边框。重要 AI 回复会显示黄色背景。文字颜色取决于最终参与层叠比较后获胜的规则。

在 DevTools 的 Styles 面板里，被覆盖的属性会出现删除线。顺着删除线往下看，通常能找到真正生效的规则。

---

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| 层叠 | CSS 机制 | 多条规则冲突时决定最终值 | 后写覆盖同优先级 |
| `!important` | 重要性标记 | 强行提高声明重要性 | `color: red !important;` |
| 书写顺序 | CSS 规则 | 同优先级时后写生效 | `.a` 后于 `.b` |
| DevTools Styles | 调试工具 | 查看样式来源和覆盖关系 | 被覆盖属性有删除线 |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 多条 CSS 规则可以同时命中同一个元素。
- 同一属性发生冲突时，浏览器按层叠规则决定最终值。
- 同优先级下，后写的规则覆盖先写的规则。

---

## ⚠️ 易错点

- 易错点1：看到类名存在就以为样式一定生效。正确写法：还要检查是否被后续规则覆盖。
- 易错点2：频繁使用 `!important`。正确写法：先理清选择器和顺序，必要时再使用。
- 易错点3：不同文件加载顺序不明确。正确写法：基础样式先加载，组件和页面样式后加载。

---

## 💡 最佳实践

- 按“基础样式 → 组件样式 → 状态样式”的顺序组织 CSS。
- 减少无意义的重复选择器，避免后续维护时难以判断谁覆盖谁。
- 调试样式冲突时，优先查看 DevTools 中最终生效规则。

---

## 🚀 AI 应用场景

AI 界面通常有基础消息样式、角色样式、状态样式、错误样式。层叠规则决定这些样式如何组合。

```css
.message {
  background: #f9fafb;
}

.message-ai {
  border: 1px solid #e5e7eb;
}

.message-error {
  background: #fef2f2;
  border-color: #ef4444;
}
```

当 AI 消息同时是错误消息时，`.message-error` 应放在后面，确保错误状态更明显。
