# 伪元素选择器（CSS-007）

## 🎯 本节学习目标

- 掌握伪元素选择器的基本写法，例如 `::before`、`::after`、`::placeholder`。
- 理解伪元素用于选择或生成元素的一部分表现内容。
- 能够用伪元素增强 AI 界面提示、标签和输入体验。

---

## 📖 什么是伪元素选择器

伪元素选择器用于选择元素的某个部分，或者在元素内部生成一个虚拟的表现层内容。常见写法是双冒号，例如 `::before`、`::after`、`::placeholder`、`::selection`。

伪元素生成的内容不是真实 DOM 节点，通常用于装饰、提示、标记和局部文本样式。

在 AI 应用中，伪元素可以给系统消息加状态标记、给引用来源加前缀、调整输入框 placeholder 的视觉弱化效果。

---

## 🧠 原理讲解

`::before` 和 `::after` 通常需要配合 `content` 属性使用：

```css
.status::before {
  content: "状态：";
}
```

浏览器会在 `.status` 元素内容前生成一段表现层文本。它能被看到，但不会像普通 HTML 文本那样成为可直接编辑的 DOM 内容。

伪元素适合做辅助视觉，不适合承载关键业务信息。关键内容仍然应该写在 HTML 中。

---

## 🏗 基本结构

```html
<style>
  .message-ai::before {
    content: "AI";
    margin-right: 8px;
  }
</style>

<p class="message-ai">这是模型回复。</p>
```

---

## ✅ 完整代码

下面用伪元素为 AI 消息和引用来源添加视觉标识：

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>伪元素选择器示例</title>
  <style>
    body {
      margin: 0;
      padding: 32px;
      font-family: system-ui, sans-serif;
      background: #f3f4f6;
      color: #111827;
    }

    .answer-card {
      max-width: 760px;
      margin: 0 auto;
      padding: 24px;
      background: #ffffff;
      border: 1px solid #e5e7eb;
      border-radius: 8px;
    }

    .message-ai {
      line-height: 1.8;
    }

    .message-ai::before {
      content: "AI";
      display: inline-block;
      margin-right: 8px;
      padding: 2px 6px;
      border-radius: 4px;
      background: #e0f2fe;
      color: #075985;
      font-size: 12px;
      font-weight: 700;
    }

    .source {
      margin-top: 16px;
      color: #4b5563;
      font-size: 14px;
    }

    .source::before {
      content: "引用 ";
      color: #6b7280;
      font-weight: 600;
    }

    .prompt-input {
      box-sizing: border-box;
      width: 100%;
      margin-top: 16px;
      padding: 12px;
      border: 1px solid #d1d5db;
      border-radius: 6px;
      font: inherit;
    }

    .prompt-input::placeholder {
      color: #9ca3af;
    }
  </style>
</head>
<body>
  <article class="answer-card">
    <p class="message-ai">CSS 伪元素可以为界面添加辅助标记，但不要承载关键内容。</p>
    <p class="source">MDN CSS Selectors</p>
    <input class="prompt-input" placeholder="继续追问这个回答">
  </article>
</body>
</html>
```

---

## 🔍 逐行解析

`.message-ai::before` 在 AI 回复前生成一个“AI”标签。

`display: inline-block` 让生成内容可以设置内边距、背景和圆角。

`.source::before` 在引用来源前生成固定前缀，减少 HTML 重复。

`.prompt-input::placeholder` 只控制输入框占位提示文字的样式。

---

## 🌐 浏览器表现

AI 回复前会出现一个浅蓝色的小标签，引用来源前会自动显示“引用”。输入框的 placeholder 会以较浅颜色展示。

在 DevTools 中可以看到 `::before` 或 `::placeholder` 规则，但它们不是普通 DOM 子节点。

---

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `::before` | 伪元素 | 在元素内容前生成内容 | `.tag::before` |
| `::after` | 伪元素 | 在元素内容后生成内容 | `.tag::after` |
| `content` | CSS 属性 | 指定生成内容 | `content: "AI";` |
| `::placeholder` | 伪元素 | 选择输入框占位文字 | `input::placeholder` |
| `::selection` | 伪元素 | 选择用户选中文本 | `::selection` |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 伪元素通常用双冒号，例如 `::before`。
- `::before` 和 `::after` 必须设置 `content` 才能显示。
- 伪元素适合辅助视觉，不适合承载必须读取的核心内容。

---

## ⚠️ 易错点

- 易错点1：忘记写 `content`。正确写法：`::before` 和 `::after` 一般都要设置 `content`。
- 易错点2：把重要文本放进伪元素。正确写法：关键信息写在 HTML 中。
- 易错点3：混淆伪类和伪元素。正确写法：状态用伪类，元素局部或生成内容用伪元素。

---

## 💡 最佳实践

- 用伪元素处理装饰性标签、分隔符、角标和占位文字样式。
- 对无障碍要求高的内容，不要依赖伪元素表达核心含义。
- AI 回答中的来源、警告、工具状态可以用真实文本加伪元素辅助增强。

---

## 🚀 AI 应用场景

AI 界面常需要展示“AI”“引用”“工具调用”“草稿”等辅助标签。伪元素可以减少重复 HTML，让视觉标记更统一。

```css
.tool-result::before {
  content: "工具";
  margin-right: 6px;
  color: #92400e;
}

.streaming::after {
  content: "";
  display: inline-block;
  width: 6px;
  height: 1em;
  background: #111827;
}
```

## 📝 练习题

1. [基础题] 使用 `::before` 给 `.note` 前面添加“提示”标签。
2. [进阶题] 使用 `::placeholder` 调整输入框占位文字颜色。
3. [AI 场景题] 为 AI 工具调用结果添加一个装饰性“工具”标记。
