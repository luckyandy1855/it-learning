# 用户主题偏好（CSS-049）

## 🎯 本节学习目标

- 理解系统偏好、用户手动选择和默认主题的关系。
- 掌握主题偏好的基本优先级设计。
- 能够为 AI 应用设计合理的主题切换策略。

---

## 📖 什么是用户主题偏好

用户主题偏好指用户希望界面使用明色、暗色或跟随系统。

一个成熟应用通常不只支持系统暗色模式，还允许用户手动选择主题，并记住这个选择。

在 AI 应用中，用户可能长时间写作、阅读和调试 Prompt。尊重主题偏好能减少疲劳，也让工具更符合个人工作环境。

---

## 🧠 原理讲解

常见优先级：

```text
用户手动选择 > 系统偏好 > 应用默认主题
```

CSS 负责主题变量和系统偏好：

```css
@media (prefers-color-scheme: dark) {
  :root {
    --bg: #0f172a;
  }
}
```

JavaScript 通常负责读取用户选择并设置 `data-theme`。本节重点是 CSS 结构和策略理解。

---

## 🏗 基本结构

```html
<html data-theme="system">
<style>
  :root {
    --bg: #ffffff;
  }

  [data-theme="dark"] {
    --bg: #111827;
  }
</style>
```

## ✅ 完整代码

下面展示支持系统偏好和手动主题的 CSS 结构：

```html
<!doctype html>
<html lang="zh-CN" data-theme="light">
<head>
  <meta charset="UTF-8">
  <title>用户主题偏好示例</title>
  <style>
    :root {
      --bg: #f8fafc;
      --surface: #ffffff;
      --text: #111827;
      --border: #e5e7eb;
    }

    @media (prefers-color-scheme: dark) {
      :root {
        --bg: #0f172a;
        --surface: #111827;
        --text: #e5e7eb;
        --border: #374151;
      }
    }

    [data-theme="light"] {
      --bg: #f8fafc;
      --surface: #ffffff;
      --text: #111827;
      --border: #e5e7eb;
    }

    [data-theme="dark"] {
      --bg: #0f172a;
      --surface: #111827;
      --text: #e5e7eb;
      --border: #374151;
    }

    * {
      box-sizing: border-box;
    }

    body {
      margin: 0;
      padding: 32px;
      font-family: system-ui, sans-serif;
      background: var(--bg);
      color: var(--text);
    }

    .settings-card {
      max-width: 680px;
      margin: 0 auto;
      padding: 24px;
      border: 1px solid var(--border);
      border-radius: 8px;
      background: var(--surface);
    }

    .theme-options {
      display: flex;
      flex-wrap: wrap;
      gap: 8px;
    }

    .theme-options button {
      padding: 8px 12px;
      border: 1px solid var(--border);
      border-radius: 6px;
      background: var(--surface);
      color: var(--text);
      font: inherit;
    }
  </style>
</head>
<body>
  <section class="settings-card">
    <h1>主题偏好</h1>
    <p>当前示例使用 data-theme="light" 强制明色主题。</p>
    <div class="theme-options">
      <button>跟随系统</button>
      <button>明色</button>
      <button>暗色</button>
    </div>
  </section>
</body>
</html>
```

---

## 🔍 逐行解析

`:root` 提供应用默认主题。

`@media (prefers-color-scheme: dark)` 提供系统暗色偏好的自动适配。

`[data-theme="light"]` 和 `[data-theme="dark"]` 表示用户手动选择，写在媒体查询后面，因此能覆盖系统偏好。

按钮只是演示 UI；真实项目需要用 JavaScript 保存用户选择。

---

## 🌐 浏览器表现

示例中 HTML 写了 `data-theme="light"`，因此即使系统是暗色，页面也会使用明色变量。

如果删除 `data-theme`，页面会回到默认变量加系统偏好的行为。

---

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `prefers-color-scheme` | 媒体特性 | 系统主题偏好 | `dark` |
| `data-theme` | HTML 属性 | 用户主题选择 | `light` |
| `localStorage` | Web API | 常用于保存偏好 | `theme=dark` |
| `matchMedia` | Web API | JS 读取媒体查询 | `matchMedia(...)` |
| cascade | CSS 机制 | 后写变量覆盖先写变量 | 用户选择覆盖系统 |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 用户手动选择通常优先于系统偏好。
- CSS 负责变量，JS 负责保存和切换用户选择。
- 主题偏好应提供“跟随系统”选项。

---

## ⚠️ 易错点

- 易错点1：手动选择无法覆盖系统暗色。正确写法：`data-theme` 规则写在媒体查询后面。
- 易错点2：只支持明暗，不支持跟随系统。正确写法：提供 system/light/dark 三种策略。
- 易错点3：页面加载时先闪白再变暗。正确写法：真实项目需在首屏渲染前尽早设置主题属性。

---

## 💡 最佳实践

- 主题选项建议为“跟随系统、明色、暗色”。
- 保存用户偏好后，页面加载早期就设置 `data-theme`。
- 主题切换控件本身也要适配主题变量。

---

## 🚀 AI 应用场景

AI 写作工具常在夜间长时间使用。用户可能希望工作区固定暗色，而不是跟随系统自动切换。
