# 类选择器（CSS-003）

## 🎯 本节学习目标

- 掌握类选择器 `.className` 的基本写法。
- 理解类名为什么是现代前端组件样式的主力方式。
- 能够用类选择器区分 AI 聊天界面的不同消息类型。

---

## 📖 什么是类选择器

类选择器通过 HTML 元素上的 `class` 属性匹配元素。CSS 中以英文句点开头，例如 `.message`、`.toolbar`、`.primary-button`。

同一个类可以被多个元素复用，一个元素也可以同时拥有多个类。这让类选择器非常适合做组件样式、状态样式和可复用 UI。

在真实前端项目里，类选择器比元素选择器更常用，因为它既能表达业务含义，又不会像 ID 选择器那样过度绑定单个元素。

---

## 🧠 原理讲解

HTML 通过 `class` 标记元素身份：

```html
<p class="message assistant">这是 AI 回复。</p>
```

CSS 通过类选择器匹配：

```css
.message {
  padding: 12px;
}

.assistant {
  background: #f9fafb;
}
```

浏览器会把 `.message` 和 `.assistant` 两条规则同时应用到这个 `<p>` 上。最终样式是多条规则共同叠加的结果。

---

## 🏗 基本结构

```html
<style>
  .message {
    padding: 12px;
    border-radius: 8px;
  }

  .user {
    background: #dbeafe;
  }
</style>

<p class="message user">请帮我总结这段文字。</p>
```

---

## ✅ 完整代码

下面用类选择器实现用户消息、AI 消息和系统提示：

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>类选择器示例</title>
  <style>
    body {
      margin: 0;
      padding: 32px;
      font-family: system-ui, sans-serif;
      background: #eef2ff;
      color: #111827;
    }

    .chat-panel {
      max-width: 720px;
      margin: 0 auto;
      padding: 24px;
      background: #ffffff;
      border-radius: 8px;
      border: 1px solid #e5e7eb;
    }

    .message {
      margin: 0 0 14px;
      padding: 12px 14px;
      border-radius: 8px;
      line-height: 1.7;
    }

    .message-user {
      background: #dbeafe;
      color: #1e3a8a;
    }

    .message-ai {
      background: #f9fafb;
      border: 1px solid #e5e7eb;
    }

    .message-system {
      background: #ecfdf5;
      color: #065f46;
      font-size: 14px;
    }
  </style>
</head>
<body>
  <section class="chat-panel">
    <p class="message message-system">系统：当前模型为 reasoning 模式。</p>
    <p class="message message-user">请把这段会议记录整理成行动项。</p>
    <p class="message message-ai">可以。我会按负责人、任务和截止时间输出清单。</p>
  </section>
</body>
</html>
```

---

## 🔍 逐行解析

`.chat-panel` 定义聊天区域的容器样式。

`.message` 抽取所有消息共有的气泡样式，包括间距、内边距、圆角和行高。

`.message-user` 表示用户消息，使用浅蓝背景和深蓝文字。

`.message-ai` 表示 AI 回复，使用浅灰背景和边框，阅读感更接近正文。

`.message-system` 表示系统状态，使用绿色提示样式，并降低字号。

---

## 🌐 浏览器表现

三个消息段落都拥有 `.message` 的共同气泡外观，但因为第二个类不同，背景色、文字色和边框会有所区别。

在 DevTools 中可以看到同一个元素被多条类规则命中。类名越清晰，调试时越容易定位样式来源。

---

## 📦 常见属性 / API

| 属性/API | 类型 | 说明 | 示例 |
|---------|------|------|------|
| `class` | HTML 属性 | 给元素添加类名 | `class="message user"` |
| `.message` | 类选择器 | 选择所有 class 含 `message` 的元素 | `.message { padding: 12px; }` |
| 多类组合 | CSS 写法 | 让一个元素叠加多组样式 | `class="message message-ai"` |
| 语义类名 | 命名方式 | 用类名表达组件含义 | `.chat-panel` |

---

## ⭐⭐⭐⭐⭐ 必学重点

- 类选择器以 `.` 开头，匹配 HTML 的 `class`。
- 一个元素可以有多个类，多个类的样式会叠加。
- 类选择器是组件化界面最常用的样式组织方式。

---

## ⚠️ 易错点

- 易错点1：CSS 写 `.message`，HTML 却写 `class="messages"`。正确写法：类名必须完全一致。
- 易错点2：类名只写颜色含义，如 `.blue`。正确写法：优先写业务含义，如 `.message-user`。
- 易错点3：把多个组件都叫 `.box`。正确写法：类名要能表达组件边界。

---

## 💡 最佳实践

- 用一个基础类承载共同样式，再用变体类区分类型。
- 类名尽量稳定，不要跟随临时视觉变化频繁改名。
- AI 界面建议使用 `.message-user`、`.message-ai`、`.message-system` 这类直观命名。

---

## 🚀 AI 应用场景

AI 聊天产品通常有多种消息：用户输入、模型回答、工具调用、错误提示、引用来源。类选择器可以把这些类型清楚地区分出来。

```css
.tool-call {
  background: #fff7ed;
  border: 1px solid #fed7aa;
}

.error-message {
  background: #fef2f2;
  color: #991b1b;
}
```
